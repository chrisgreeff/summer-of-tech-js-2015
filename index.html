Loosely Typed

Before I move onto move details about the types, I should mention that JavaScript is a loosely typed language. So unlike Java or C, you don't have to be explicit in the type of variable you are declaring. You can just use the keyword `var`, and JavaScript will work out what you're talking about.

Ok, now lets move onto functions. There are 2 ways to create functions. Firstly with the keyword function and then the name. Or assigning it to a `var`. Invoking the function is similar to how you would in most languages, with brackets, and the params passed in.

On of the harder concepts to grasp in JS is their implementations of scopes. So in JS there is function scope. Meaning everything within the function brackets have access to everything declared within the function, and outside. Take this for example.

Functions


var a = 'outer';

function myFunc() {
  var b = 'inner';

  console.log(a); // 'outer'
  console.log(b); // 'inner'
}

myFunc();

console.log(a); // 'outer'
console.log(b); // ERROR
Here we can see the function has no trouble logging both a and b, since they are all in scope. However the outer console logs has no access to the variables declared within the function, even though it has been invoked. This is cool though! We're happy with this.

So another example. In this case, when we call `changeValueOfA`, since we are manipulating `a` which is in the same scope, we should be able to change the value right?

Functions


var a = 'old';

function changeValueOfA() {
  a = 'new';
  return;

  var a;
}

console.log(a); // 'old'
changeValueOfA();
console.log(a); // 'old'
Nope! So what's actually going on here? Well this is something called variable hoisting. And what's happening, is right before runtime, the JavaScript compiler actually processes this code block like this. Can you see the difference?

Functions


var a = 'old';

function changeValueOfA() {
  var a;
  a = 'new';

  return;
}

console.log(a); // 'old'
changeValueOfA();
console.log(a); // 'old'
The function declaration of `a` is hoisted to the top of the function scope. So now all of a sudden the `a` we thought we were manipulating is now the locally scoped function declaration. And due to the scope concept we were talking about previously, this leaves the outer variable untouched. Problems like this can leave us developers feeling like this.

var obj = {};

Ok lets take a look at Objects. Essentially the easiest way to think of objects in JavaScript, is hashmaps in Java, or associative arrays in PHP. So key value pair. There are two ways to create simple objects. Although you should almost never use the second way.

Objects


var danceBot = {
  name: 'DanceBot',
  'my version': 5000
};

danceBot.dance = function() {
  console.log('Boogie boogie, get down tonight!')
};
So here, I am declaring an object, with predefined properties of any type, strings, booleans, whatever. And I can also add properties at anytime bu just assigning it. So here I am assigning a function.

Objects


function Person(name, age, lovesToDance) {
  this.name = name;
  this.age = age;
  this.lovesToDance = lovesToDance;
}
JavaScript is also an Object Oriented language, so we can create instances of objects like this. Declaring a constructor as a function, and assigning properties to the instance with the `this` keyword. So in this example, we have a function named Person. To create an instance, we just need the `new` keyword before the function, and JavaScript knows to return an object.

Another special type of object are arrays. Similarly there are 2 ways to create arrays.

Arrays


var arr = ['dog', 'cat', 'hen'];

You can assign values to the array like this. So predefining values into the array on declaration. And then pushing values to the stack using the `push` function.

Arrays


var arr = ['dog', 'cat', 'hen'];

arr[100] = 'mouse';
arr.length; // ?
How about if I do this? What should happen?

Arrays


var arr = ['dog', 'cat', 'hen'];

arr[100] = 'mouse';
arr.length; // 101
Not what you expect...

'dog','cat','hen',,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,'mouse'

So what's going on?? JavaScript has assigned the 100'th element in the array with 'mouse', but also taken the liberty of assigning all elements of the array with undefined. Thanks JavaScript!
